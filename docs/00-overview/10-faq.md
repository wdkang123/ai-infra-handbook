# 常见问题 FAQ

## 这套仓库现在到底是文档项目，还是代码项目

两者都是，但主目标是学习手册优先。

更准确地说，它现在是一套：

- 以文档为主线
- 以可运行项目为学习支撑

的学习型工程仓库。

## 现在这些代码是“真实代码”吗

是学习型真实代码，不是完整生产代码。

也就是说：

- 它们是真的能跑
- 也能形成最小联调闭环
- 但没有一开始就把所有复杂度都做满

这正是为了让你后面更容易边学边改。

## 我是不是必须先把所有文档看完才能开始动手

不是。  
更推荐的是：

1. 先看总览
2. 再跑最小手册
3. 然后做第一次 walkthrough
4. 再回头读对应章节

也就是“跑起来”和“看明白”交替进行。

## 我是不是应该一开始就把代码改得很真实

不建议。

当前更适合的节奏是：

- 先理解结构
- 先跑通主链
- 再做单点小改动
- 之后再逐步把某一层变得更真实

## 如果我跑通了，但还是觉得很多地方是 mock，这正常吗

很正常。  
因为这套仓库本来就是为了支持“先搭骨架，再逐步替换内部实现”的学习路径。

如果你已经能看懂骨架，那就已经走在正确方向上了。

## 后面我可以让别的大模型继续帮我细化代码吗

完全可以，而且这是预期用法之一。

更好的方式通常是：

- 先用这套手册建立结构理解
- 再在具体卡点上，用别的大模型帮你细化实现或解报错

这样效率会比完全从零问一个模型更高。

## 我怎么判断自己是不是学明白了一点

最简单的方式不是问“我看了多少页”，而是问：

- 我能不能说清楚四层系统地图
- 我能不能跑通最小闭环
- 我能不能分清 gateway 和 inference-service 的边界
- 我能不能理解 run / compare / history 为什么存在

如果想更系统一点，就直接看：

- [学习检查点](/00-overview/09-learning-checkpoints)

如果准备公开分享，再做：

- [公开发布验收 Lab](/07-hands-on-labs/06-public-release-readiness-lab)

## 什么时候算真正可以进入下一阶段

如果你已经能：

- 跑通最小闭环
- 挑一条主线连续读下去
- 做一个小改动再重跑
- 大概知道自己卡点属于哪一层

那就已经可以进入“边学边改、边跑边细化”的下一阶段了。

## 这一页学完应该带走什么

这套仓库不要求你一开始就把所有东西都学全。  
它更像一条渐进路线：先建立系统感，再逐步把实现变真。
