# 常见学习误区

## 为什么这一页要放在总览里

因为 AI Infra 很容易出现一种表面上的“很努力”，但实际上学习路径并不高效。

这通常不是因为你不认真，而是因为这个领域天然容易让人：

- 一下看太多组件
- 一下追太多框架
- 一下想把代码做得特别真实

所以先把常见误区点出来，反而会更省时间。

## 误区一：一开始就想把所有源码看完

这通常不是最高效的起点。

更好的方式通常是：

1. 先跑起来
2. 看输出物
3. 再带着问题回看关键文件

因为 AI Infra 里很多代码，如果脱离上下文直接读，会非常像“复杂实现细节的堆叠”。

## 误区二：一开始就追求生产级真实实现

这也是最常见的坑之一。

当前仓库里很多模块是“学习型实现”，不是完整生产系统。  
这不是弱，而是刻意控制复杂度。

如果你太早把目标变成：

- 全部真实后端
- 全部真实训练
- 全部真实平台策略

学习成本会陡增。

## 误区三：把 gateway 和 inference-service 混在一起看

这会让你很难建立系统边界。

更稳的做法是先明确：

- inference-service 更像执行层
- ai-gateway 更像治理层

这条边界一旦清楚，后面很多内容都会顺很多。

## 误区四：只看 benchmark，不看上下文

benchmark 很重要，但它不是全部。

更完整的判断通常要一起看：

- benchmark
- run / compare
- observability
- 成本与延迟

否则很容易把“某个榜单分数”误当成最终结论。

## 误区五：一上来就决定必须微调

训练当然重要，但不是每个问题都应该先走微调。

很多时候更该先问：

- prompt / workflow 是否已经稳定
- 是否更像知识接入问题
- 是否已经有最小评测路径

把这些问题想清楚，再进入训练会更稳。

## 误区六：只读文档，不回代码

也不够。  
因为这套仓库本来就不是纯文档站。

最好的节奏通常是：

- 文档给你地图
- 代码给你当前实现
- 命令和结果给你反馈

三者一起看，理解会长得更快。

## 误区七：跑通一次就停

第一次跑通只是起点。  
更有价值的下一步通常是：

- 选一条主线
- 改一个小点
- 再跑一轮
- 观察结果变化

这样你才会真正开始建立系统直觉。

## 这一页学完应该带走什么

AI Infra 学习最重要的不是一开始学得多快，而是路径要稳。  
避开这些常见误区之后，你后面再继续深入，速度反而会更快。
