# 鉴权、路由、限流

## 为什么平台层不能只做“转发”

如果 gateway 只是一个透明代理，那它其实很难承担平台职责。  
平台层最少要回答三件事：

1. 谁可以进来
2. 这条请求该去哪
3. 这条请求能不能在当前频率下被接受

## 鉴权

鉴权先解决的是“是不是合法请求”。  
这件事不只是安全问题，也是在保护下游服务资源。

在当前仓库里，你会先看到最小 Bearer token 方案。  
这不是最终形态，但足够让你理解：平台层和模型服务层的边界从这里开始分开。

## 路由

路由解决的是“同一个外部模型名，最后到底落到哪个下游执行目标”。  
这就是为什么 gateway 层经常会有：

- 外部模型名
- 下游 target model
- downstream base URL

学习时最重要的不是复杂路由策略，而是先理解：

- 路由本身就是平台职责
- 模型服务本体不一定知道平台外部命名方式

## 限流

限流的核心不是“让系统显得严格”，而是保护系统稳定性。  
如果没有这一层，下游服务很容易在高峰时被击穿。

学习阶段，你先看最小内存限流就够了。重点不是算法复杂度，而是理解：

- 限流为什么通常放在平台层
- 它和鉴权、路由一样，都是入口治理的一部分
