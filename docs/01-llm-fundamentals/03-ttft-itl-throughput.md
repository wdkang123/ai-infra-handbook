# TTFT、ITL、吞吐

## TTFT 是什么

TTFT 可以先理解成“从请求进来到第一个 token 出来的时间”。  
它特别影响用户体感，因为用户会先经历一段“还没开始回答”的等待。

如果 TTFT 很高，用户经常会觉得：

- 服务卡住了
- 模型没有开始响应
- 平台很重

## ITL 是什么

ITL 可以先理解成“生成阶段相邻 token 之间的间隔”。  
如果 TTFT 决定“多久开始说话”，那 ITL 更像决定“说话流不流畅”。

所以：

- TTFT 更像启动等待
- ITL 更像输出节奏

## Throughput 是什么

throughput 关心的是单位时间内系统总体能处理多少工作量。  
它通常和并发、batch、显卡利用率、队列调度有关。

换句话说：

- TTFT 更偏单请求体验
- ITL 更偏单请求流畅度
- throughput 更偏系统整体产能

## 为什么三者经常冲突

你后面会反复看到一个现实：  
优化吞吐，不一定会优化用户体验；优化单请求体验，也不一定会优化总体产能。

比如：

- 更激进的 batching 可能提高吞吐，但让 TTFT 变差
- 更追求低延迟可能让资源利用率下降

所以 AI Infra 里的很多设计，本质上不是“对错题”，而是“取舍题”。

## 学这一节最重要的直觉

以后看到性能讨论时，先问清楚对方到底在说哪一个：

1. TTFT
2. ITL
3. throughput

如果这三个词没分开，很多性能结论其实很容易说混。
