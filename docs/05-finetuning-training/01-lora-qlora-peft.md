# LoRA、QLoRA、PEFT

## 为什么这几个词总会一起出现

因为它们都属于“怎么更现实地做微调”这条主线。

如果每次都全量更新整个模型参数，成本通常太高。  
所以工程里才会不断出现更轻的微调路线。

## LoRA

LoRA 可以先理解成“不是整模型全改，而是只引入一小部分可训练增量”。  
学习时最重要的不是公式，而是直觉：

- 训练更轻
- 保存更轻
- 部署时也更容易管理增量部分

## QLoRA

QLoRA 可以先理解成“在更节省资源的前提下继续走轻量微调”。  
它让“资源不那么豪华”的场景里也更容易做实验。

## PEFT

PEFT 更像一个更大的视角：  
LoRA、QLoRA 这些都可以放到“参数高效微调”这类方法里理解。

所以学习时先抓住这一层就够了：

- 目标不是全量训练
- 目标是用更小代价把模型往任务方向推一段

## 对当前仓库的意义

当前 `finetune-demo` 不是完整训练框架，但它已经把学习最重要的结构收出来了：

- train
- save
- export
- run artifacts
- history

所以它更适合作为“理解训练工程资产长什么样”的入口。
