- 我要跑哪个命令

- 某个输出文件是什么意思

## 怎么使用

如果你是第一次学习，不建议从这里开始。  
先看 [课程大纲](/00-overview/12-course-syllabus) 或 [第一次实操演练](/00-overview/04-first-walkthrough)。

如果你已经跑过一轮，这组页面会更有用。
如果你已经看到具体 header、events、JSON 或 manifest，但不知道怎么解释，优先看 [示例输出与证据库](/13-output-gallery/00-overview)。
如果你准备维护课程结构、发到 GitHub 或组织共学，优先看 [学习站清单生成器](/09-reference/08-learning-inventory)。
如果你准备把这些页面组织成一次可带练课程，优先看 [课程目录生成器](/09-reference/10-course-catalog)。
如果你准备做 GitHub 首发、PR 复盘或公开展示，优先看 [发布摘要生成器](/09-reference/09-release-brief)。
如果你准备直接安排共学议程、分发模块任务或收集课后复盘，优先看 [自动生成共学包](/14-workshop-kit/07-generated-workshop-packet)。
如果你准备按模块出题、评分或做 PR review，优先看 [自动生成测评包](/10-assessments/06-generated-assessment-pack)。
如果你准备把首批路线图拆成 GitHub issue，优先看 [自动生成路线图包](/08-publication/05-generated-roadmap-pack)。
如果你准备创建第一个 release、首批 issue 或发布后复盘，优先看 [自动生成首发运营包](/08-publication/13-generated-launch-pack)。

## 参考资料的维护原则

每个条目尽量包含：

- 入口命令或文件路径
- 它解决的问题
- 应该观察的结果
- 相关学习页面
- 需要跑的验证命令

这样读者不需要在文档和项目之间来回猜。
