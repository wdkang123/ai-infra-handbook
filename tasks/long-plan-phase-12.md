# Long Plan Phase 12

## 名称

Codex Task Pack Phase

## 目标

在 execution slices 收口后，把下一层文档继续压成可直接发给 Codex 的实现任务卡包。

## 本阶段重点

1. 为每个子项目生成首轮 Codex 实现任务卡
2. 为根级联调生成跨项目任务卡
3. 把“可以开始写代码”的入口进一步模板化

## 本阶段不做

- 不直接写仓库源码
- 不重开架构讨论
- 不扩 MVP 范围
- 不修改 accepted 资产

## 预期结果

- 每个项目都有第一批可执行的 Codex 任务卡
- 每张卡都能独立说明输入资产、目标文件、验收命令与 cut line
- 根级联调也有自己的任务卡和切换门槛
