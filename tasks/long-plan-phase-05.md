# Long Plan Phase 05

目标：把 MiniMax 的长跑模式继续升级成“深研究 0 接管模式”，进一步拉长单次运行时间，并减少用户回合式分发。

## 为什么 13 分钟还是偏短

上一轮 0 接管链已经把文件数拉到 28 个，但仍然偏快，原因主要有三点：

- 仍以仓库内已有资产重组为主
- 很多文件是索引与总结，研究深度有限
- 对官方 release / changelog / 边界核对要求还不够强

这说明当前问题不再是“包太少”，而是“包不够重”。

## 本阶段升级方向

从“专题包串联”升级到“深研究专题链”：

- 每包 8 个左右交付物
- 每包强制包含至少一个边界澄清类文件
- 每包强制包含至少一个 timeline / map / memo 类文件
- 整体更强调官方来源、版本入口、更新线索和系统化映射

## 适合深研究的五条链

1. Inference Engines 深研究链
2. Observability / Evaluation 深研究链
3. Finetuning / Training 深研究链
4. Cross-Project Systemization 深研究链
5. Execution Decomposition 深研究链

## 目标效果

让用户只需要：

1. 启动一次深研究运行
2. 长时间离开
3. 回来后批量验收和收口

而不是每隔十来分钟重新投喂下一轮。
