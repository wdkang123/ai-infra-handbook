# Long Plan Phase 11

## 名称

Execution Slice Phase

## 目标

在 starter blueprints、fixtures、implementation maps 全部收口后，继续把项目文档收敛成可直接服务 Codex 真正开写代码的 execution slices。

## 本阶段重点

1. 为四个子项目分别产出 execution slice 顺序与切片契约
2. 为根级联调产出跨项目切片和切换门槛
3. 把“可以开始写代码”前的最后一层文档抽象压到最低

## 本阶段不做

- 不直接写仓库源码
- 不重开架构讨论
- 不扩 MVP 范围
- 不改 accepted 蓝图

## 预期结果

- 每个项目都有一套可直接驱动 Codex 的 slice order
- 每个 slice 都有明确的入口文件、验证门槛和 cut line
- 根级联调也有自己的切片与集成门槛
